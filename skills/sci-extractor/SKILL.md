# 变异文献提取工具集

本 Skill 包含两个互补的文献提取工具，根据不同数据源选择使用。

| 工具 | 脚本 | 数据源 | 适用场景 |
|------|------|--------|----------|
| **PubMed 提取器** | `scripts/pubmed_variant_extractor.py` | PubMed 摘要 (E-utilities API) | 已有 PMID 引用网络，需快速汇总变异证据 |
| **PDF 提取器** | `scripts/sci_extractor.py` | 本地 PDF 全文 | 有 PDF 全文文件，需深入提取表格和患者信息 |

# 一、PubMed 摘要变异提取器

从 PubMed 引用网络中批量获取文献摘要，提取目标变异的证据信息，输出 **JSON + GBK CSV + XLSX** 三种格式。

**脚本路径**: `scripts/pubmed_variant_extractor.py` — 所有逻辑预置在脚本中，仅需修改配置即可运行，确保每次稳定可重复。

## 环境准备（PubMed 提取器，首次使用执行一次）

```bash
pip install openpyxl
```

## 使用方法（PubMed 提取器）

### 两步完成

**步骤 1** — 编辑 `scripts/pubmed_variant_extractor.py` 第 30-51 行，填入目标变异和 PMID 列表：

```python
# ==================== 目标变异配置 ====================
GENE = "CFTR"                    # 基因名（HGNC 标准名）
TRANSCRIPT = "NM_000492.4"       # 参考转录本
CDNA = "c.613C>T"                # cDNA 改变（HGVS 命名）
PROTEIN = "p.Pro205Ser"          # 蛋白改变（三字母）
EXON = "exon 6"                  # 外显子位置

# PMID 列表：以 [PMID:xxxxx] 开头的行为主文献，后续行为其引用网络
# 格式：[PMID:主文献]|引用1|引用2|...
PMID_GROUPS_TEXT = """
[PMID:7505694]|18456578|21097845|23974870|31036917|31589614|34782259
[PMID:23974870]|10225950|11938353|16114821|21097845|23857699|24583165|25363320|28603918|31036917|34065744|34296343|34782259|36834620|7505694|8663008|9920885
"""

# 输出目录
OUTPUT_DIR = r"D:\claude_code\project1\文献提取结果"
# ======================================================
```

**步骤 2** — 一行命令运行：

```bash
python .claude/skills/sci-extractor/scripts/pubmed_variant_extractor.py
```

脚本自动完成：
1. 解析 PMID 列表并去重
2. 通过 NCBI E-utilities API 批量获取摘要（每批 10 篇，0.35s 间隔）
3. 分析每篇文献中目标变异的提及情况（多关键词匹配）
4. 输出三种格式文件

### 输出文件

| 输出类型 | 文件命名 | 编码 | 说明 |
|----------|----------|------|------|
| **JSON** | `{GENE}_{CDNA}.json` | UTF-8 | 完整结构化数据，含摘要、作者、期刊、关键词匹配等 |
| **CSV** | `{GENE}_{CDNA}_文献汇总.csv` | GBK, tab 分隔 | 10 列标准化表格，与现有文件格式一致 |
| **XLSX** | `{GENE}_{CDNA}_文献汇总.xlsx` | n/a | 带格式表格，冻结表头、自动筛选 |

### CSV/XLSX 表格列说明

| 列 | 说明 |
|----|------|
| PMID | PubMed ID |
| 标题 | 文献标题 |
| 是否提及此位点 | `是 — 匹配: <关键词>` 或 `否（<一句话概括>...）` |
| 患者数 | 从摘要自动提取（如 `n=123` 模式） |
| 致病性 | 从摘要自动提取致病性关键词 |
| 关联合子状态 | 复合杂合 / 纯合 / 杂合 / 取决于具体基因型 |
| 反式(trans)位点 | 从摘要提取共存变异（如 F508del, G551D） |
| 患者临床表型 | 从摘要提取表型关键词（囊性纤维化、胰腺功能等） |
| 文献背景(是什么研究) | 一句话概括 + 期刊年份 |
| 总结 | 相关句子摘要 + 参考文献信息 |

### 变异关键词匹配

脚本基于 CDNA 和 PROTEIN 配置自动生成多级关键词：

| 来源 | 生成规则 | 示例（c.613C>T, p.Pro205Ser） |
|------|----------|------|
| CDNA 原值 | 直接使用 | `c.613C>T` |
| CDNA 简写 | `c.` 后的数字+碱基变化 | `613C>T` |
| PROTEIN 原值 | 直接使用 | `p.Pro205Ser` |
| PROTEIN 去 p. | 去掉 `p.` 前缀 | `Pro205Ser` |
| 缩写形式 | 单字母氨基酸简写 | `P205S` |
| 残基编号 | 仅残基编号+名称 | `Pro205` |

匹配大小写不敏感，确保同一变异的不同命名形式均能被捕获。

### 稳定性保证

| 保证项 | 实现方式 |
|--------|----------|
| 预置配置 | 所有参数写死在脚本顶部，用户只改变异和 PMID 列表 |
| 确定性输出 | 同一配置 + 同一 PMID 列表 → 永远相同的 JSON/CSV/XLSX |
| PMID 去重 | 自动去重，主文献优先排序 |
| 编码一致性 | CSV 使用 GBK（与现有文件一致），JSON/XLSX 使用 UTF-8 |
| API 容错 | 单批失败不中断，继续处理剩余批次 |
| 速率控制 | 每批 0.35s 间隔，符合 NCBI E-utilities 使用规范 |

# 二、PDF 全文变异提取器

从本地 SCI 文献 PDF 中提取目标变异的结构化信息，生成标准化中文总结段落，结果输出为 Excel 表格。

**脚本路径**: `scripts/sci_extractor.py` — 所有逻辑预置在脚本中，仅需修改配置即可运行，确保每次稳定可重复。

## 环境准备（PDF 提取器，首次使用执行一次）

```bash
pip install PyMuPDF openpyxl
```

## 使用方法（PDF 提取器）

### 两步完成

**步骤 1** — 编辑 `scripts/sci_extractor.py` 第 32-40 行，填入目标变异：

```python
GENE = "ABCA4"                    # 基因名（HGNC 标准名）
CDNA = "c.2536G>C"                # cDNA 改变（HGVS 命名）
PROTEIN = "p.Asp846His"           # 蛋白改变（三字母）
PROTEIN_SHORT = "p.D846H"         # 蛋白改变（单字母简写）
EXTRA_KEYWORDS = [                # 前几个为变异特异性词，后几个为泛化词
    "c.2536", "2536G>C", "D846H", "Asp846His", "846His",
    "p.Asp846", "p.D846",
    "ABCA4", "NM_000350", "Stargardt", "retinal", "retina"
]
```

> **配置原则**：`EXTRA_KEYWORDS` 靠前放变异特异性词（`c.2536`、`D846H` 等），靠后放基因名等泛化词（`ABCA4`、`retinal` 等）。特异性词用于患者精确计数，泛化词用于全文句子搜索。

**步骤 2** — 一行命令运行：

```bash
python .claude/skills/sci-extractor/scripts/sci_extractor.py
```

脚本自动扫描 `sci/` 目录下所有 PDF，提取全文和表格，搜索目标变异并生成结果。

### 输出文件

| 输出类型 | 默认路径 | 说明 |
|----------|----------|------|
| **Excel** | `D:\claude_code\project1\文件提取结果\{GENE}_{CDNA}_提取结果.xlsx` | 主要输出，带格式表格 |
| JSON | `sci/sci_variant_results.json` | 结构化完整数据 |

### Excel 表格表头

| 列 | 说明 |
|----|------|
| 文件名 | PDF 文件名 |
| 是否提及此位点 | 是 / 否 |
| 患者数 | 携带目标变异的患者数量（特异性关键词匹配 + 患者 ID 去重；未找到时为 0） |
| 致病性 | 针对目标变异的致病性评级 |
| 关联合子状态 | 杂合 / 纯合 / 复合杂合 |
| 反式(trans)位点 | 是否有明确父母验证 / trans 证据 |
| 患者临床表型 | 五级策略提取的患者表型 |
| 文献背景(是什么研究) | 变异提及时取首个相关句；未提及时自动生成约 100 字文献主题简介 |
| 总结 | 标准化中文描述段落（核心输出，仅含文献实际信息） |

### 高级用法

```bash
# 指定 PDF 目录
python .claude/skills/sci-extractor/scripts/sci_extractor.py --sci-dir D:\path\to\pdfs

# 指定 Excel 输出目录
python .claude/skills/sci-extractor/scripts/sci_extractor.py --excel-dir D:\my_results

# 仅输出 JSON，不生成 Excel
python .claude/skills/sci-extractor/scripts/sci_extractor.py --no-excel

# 组合使用
python .claude/skills/sci-extractor/scripts/sci_extractor.py \
    --sci-dir D:\path\to\pdfs --excel-dir D:\my_results
```

| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `--sci-dir` | 否 | `sci/` | PDF 文件目录 |
| `--output` | 否 | `sci/sci_variant_results.json` | JSON 输出路径 |
| `--excel-dir` | 否 | `D:\claude_code\project1\文件提取结果` | Excel 输出目录 |
| `--no-excel` | 否 | 关闭 | 不生成 Excel，仅输出 JSON |

## 工作流程（全自动）

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 扫描 PDF | 读取目标目录下所有 `.pdf` 文件 |
| 2 | 文本提取 | 三级回退链：PyMuPDF → pypdf + pdfplumber → 仅 pypdf |
| 3 | 关键词扩展 | 从配置字段自动生成特异性 + 泛化两级关键词 |
| 4 | 全文搜索 | 泛化关键词在全文句子级搜索目标变异上下文 |
| 5 | 患者精确定位 | 特异性关键词在表格行中精确定位携带该变异的患者 |
| 6 | 表型提取 | 五级策略：表格行 → 关键词行 → 患者描述段落 → 患者编号+变异行 → 正文表格文本行 |
| 7 | 特征提取 | 共存变异、实验室指标、变异特征（CpG/NMD/结构域/转染等） |
| 8 | 总结生成 | 根据遗传模式和文献实际数据，套用预置模板生成标准化中文段落 |
| 9 | 结果输出 | 控制台打印摘要 + JSON + **Excel 表格** |

## 核心设计：两级关键词匹配

| 关键词类型 | 来源 | 示例 | 用途 |
|------------|------|------|------|
| **特异性关键词** | `EXTRA_KEYWORDS` 中匹配变异命名模式的词 | `c.2536G>C`、`p.D846H`、`2536G>C` | 表格行级匹配 + 患者计数 |
| **泛化关键词** | `EXTRA_KEYWORDS` 全部词 + 基因名 | `ABCA4`、`NM_000350`、`retinal` | 全文句子级搜索 |

特异性关键词由正则 `^(c\.\d+|p\.\w|\d+[A-Z]>[A-Z]|[A-Z]\d+[A-Z])` 从 `EXTRA_KEYWORDS` 中自动筛选。两级分离确保精确性和召回率平衡。

### 变异命名变体匹配

使用子串匹配（`in` 操作符），同一变异的不同命名形式均能被捕获：

| 目标变异 | 能匹配的文献表述 |
|----------|-----------------|
| `c.6239dup` | `c.6239dup`、`c.6239dupA`、`6239dupA`、`c.6239_6239dup` |
| `c.565_567del` | `c.565_567del`、`565_567del`、`565del`、`c.565del` |
| `c.5461-10T>C` | `c.5461-10T>C`、`5461-10T>C`、`IVS38-10T>C` |

## 文本提取回退链

| 优先级 | 工具组合 | 触发条件 | 输出 |
|--------|----------|----------|------|
| 1 | PyMuPDF (fitz) | 默认 | 全文文本 + 结构化表格 |
| 2 | pypdf + pdfplumber | PyMuPDF 可读率 < 50% | pypdf 文本 + pdfplumber 表格 |
| 3 | 仅 pypdf | pdfplumber 未安装 | 仅文本，无表格 |
| 失败 | — | 所有方法均失败 | 错误提示：嵌入字体/扫描版 |

## 总结段落生成原则

脚本根据遗传模式和文献实际数据生成段落，核心原则：

- **仅输出文献中确实存在的信息**，不杜撰任何内容
- 无反式位置证据 → 不生成相关句子
- 无功能验证数据 → 不添加"尚未经过功能学验证"
- 无共分离数据 → 不生成相关句子
- 共存变异仅在目标变异同一行/段落中出现时才被提取

### 隐性遗传模板（复合杂合/纯合）

```
X例XX患者为该c.XXX变异与一个致病性或可能致病性c.XXX变异的复合杂合
该先证者通过父母检测确认处于反式位置（仅当文献有trans证据时添加）
该变异为本文首次报道的新型突变（仅当文献提到novel/first report时）
该突变发生于CpG二核苷酸区域（仅当文献提到CpG时）
该患者表型为xx、xx、xx等
实验室检查示xx（如有实验室检查值）
该突变虽产生提前终止密码子，但经实验验证未引发无义介导的mRNA降解（仅当NMD阴性证据时）
截短后的蛋白缺失XX结合域（仅当有结构域影响证据时）
转染野生型XX微型基因后XX能力提升（仅当有转染实验证据时）
参考文献：xxx
```

## 未提及变异时的文献简介

当文献**未提及**目标变异时，脚本自动从标题和正文中提取关键主题词（基于 140+ 词条 `TOPIC_KEYWORD_MAP`），并抓取摘要/引言正文，生成约 **100 字中文文献简介**。

示例：
- `该文献为Stargardt病、视网膜、基因型-表型相关研究。主要内容：Stargardt disease is an autosomal recessive retinal degenerative disorder...`
- `该文献为全外显子组测序、遗传性视网膜病、中国人群相关研究。对1243例中国疑似RP患者进行基因检测，诊断率72.08%。`

## 稳定性保证

| 保证项 | 实现方式 |
|--------|----------|
| 预置脚本，零编码 | 所有逻辑写死在 `sci_extractor.py`，用户只改配置字段 |
| 两级关键词匹配 | 特异性关键词精确计数，泛化关键词广泛搜索，互不干扰 |
| 确定性逻辑 | 所有提取均为确定性规则匹配，同一 PDF + 同一配置 → 永远相同输出 |
| 变异类型判断 | 基于 PROTEIN 字段中 Ter/X/Stop/fs/del/ins 直接判定 |
| 遗传方式投票 | 每个变异出现位置 ±250 字符内独立判断后投票决定 |
| 患者计数去重 | 仅当表格行含特异性关键词才计入，按患者 ID 去重；未找到变异时返回 0 |
| Excel 持久化 | 每次运行覆盖输出 Excel + JSON，结果可审计、可对比 |

## 注意事项

- PDF 使用嵌入字体（无 Unicode 映射）或为扫描版时，所有工具均无法读取文字 → 建议先 OCR 处理
- 患者数量基于特异性关键词匹配 + 患者 ID 去重，不会因基因名命中而误计
- 总结段落为脚本根据原文数据自动生成，不添加原文未提及的信息
- 所有功能变更直接修改 `scripts/sci_extractor.py`，修改后运行相同命令即可
